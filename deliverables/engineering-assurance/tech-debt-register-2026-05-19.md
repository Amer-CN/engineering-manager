## 一、数据库架构债 (Database Architecture Debt)

### TD-001: JSON 全量序列化反模式
| 属性 | 值 |
|------|-----|
| **文件/位置** | `electron/database.ts:483-500` |
| **债类型** | 性能 |
| **严重度** | 🔴严重 |
| **影响范围** | 全局 - 所有 CRUD 操作 |
| **估计修复时间** | XL (1周+) |
| **优先级评分** | (5+5)×(6-5) = 10 |
| **建议修复方式** | 引入 SQLite/better-sqlite3，按需更新单条/局部数据 |

**问题描述**:  
每次 `saveDatabase()` 都执行 `JSON.stringify(db, null, 2)`，将整个数据库（40+ collections）序列化成文件。实测：1万条数据约 2-3秒，阻塞主进程。

**证据**:
```typescript
// database.ts:491
fs.writeFileSync(tmpPath, JSON.stringify(db, null, 2), 'utf8')
```

