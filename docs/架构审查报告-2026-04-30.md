## 五、集成决策

### 🎯 审计日志集成策略

**推荐方案**: AOP 风格拦截

```typescript
// 方式 1: Hook 封装 (推荐)
export function useAuditedMutation<T>(mutation: () => Promise<T>, resource: string) {
  return async (...args) => {
    const result = await mutation(...args)
    if (result.success) {
      logCreate(resource, result.data)
    }
    return result
  }
}

// 方式 2: 组件级别手动调用
const handleSubmit = async () => {
  const result = await createPartner(data)
  if (result.success) {
    logCreate('partners', { id: result.data.id, name: data.name })
  }
}
```

## 六、架构决策记录 (ADR)

### ADR-003: 新功能集成策略

**状态**: Proposed

**背景**:
审计日志、权限系统、导入导出功能已实现，但未集成到业务组件。

**决策**:
采用渐进式集成策略:

1. **第一阶段**: 审计日志集成 (优先级: 高)
   - 在所有 CRUD 操作中添加日志记录
   - 覆盖: Members, Partners, Invoices, Projects, Contracts

2. **第二阶段**: 权限系统集成 (优先级: 中)
   - 在关键操作按钮上添加权限检查
   - 覆盖: 删除、创建、审批等敏感操作

3. **第三阶段**: 导入导出集成 (优先级: 中)
   - 在列表页面添加工具栏
   - 覆盖: 所有列表页面

**后果**:
- ✅ 所有操作都有审计追踪
- ✅ 权限控制精细到按钮级别
- ✅ 数据可批量导入导出
- ⚠️ 需要修改多个业务组件
