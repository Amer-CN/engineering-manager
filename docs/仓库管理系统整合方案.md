# 材料与进销存整合设计方案

## 一、设计背景

### 现状分析

| 模块 | 现有材料管理 | 现有进销存 |
|------|-------------|-----------|
| **定位** | 项目部材料管理 | 商贸公司采购销售 |
| **核心功能** | 采购记录、库存统计 | 采购入库、销售出库、库存预警 |
| **库存归属** | 项目级（无实时库存） | 公司级（实时库存） |
| **流向** | 采购 → 项目使用 | 采购入库 → 销售出库 |
| **关注点** | 项目成本 | 公司经营流水 |

### 痛点
- 供应商、商品档案重复维护
- 项目材料与公司库存割裂
- 无法支持公司库存调拨给项目
- 数据分析维度单一

---

## 二、整合思路

### 核心概念重构

用 **"仓库"** 作为核心抽象，统一管理所有物资：

```
                    ┌─────────────────────────────────────┐
                    │         统一商品/物料档案库            │
                    │  (名称、编码、分类、单位、规格、价格)   │
                    └─────────────────────────────────────┘
                                    ▲
                                    │ 关联
        ┌───────────────────────────┼───────────────────────────┐
        │                           │                           │
        ▼                           ▼                           ▼
┌───────────────┐         ┌───────────────┐         ┌───────────────┐
│  项目仓库      │         │  公司仓库A    │         │  公司仓库B    │
│ (项目自用)     │◄─调拨──►│ (商贸库存)   │◄─调拨──►│ (商贸库存)   │
└───────────────┘         └───────────────┘         └───────────────┘
        │                           │                           │
        ▼                           ▼                           ▼
   项目领料出库               采购入库/销售出库             采购入库/销售出库
```

### 设计原则
1. **一套数据，多种视图** - 商品档案统一，出入库记录按仓库区分
2. **灵活归属** - 物资可以归属项目，也可以归属公司仓库
3. **支持调拨** - 公司库存可调拨给项目，项目剩余物资可退回公司
4. **成本穿透** - 项目领料自动记录成本，便于成本核算

## 三、数据模型设计

### 1. 仓库 (Warehouse)

```typescript
interface Warehouse {
  id: number
  name: string                    // 仓库名称
  type: 'project' | 'company'     // 仓库类型
  projectId?: number              // 关联项目（项目仓库必填）
  location?: string               // 仓库地址
  manager?: string                // 负责人
  remarks: string
  createdAt: string
}
```

### 2. 商品/物料档案 (Product)

```typescript
interface Product {
  id: number
  code: string                    // 商品编码
  name: string                    // 商品名称
  category: string                // 分类（钢材、水泥、五金...）
  unit: string                    // 计量单位
  specifications: string          // 规格型号
  
  // 价格信息
  purchasePrice: number           // 采购参考价
  salePrice: number               // 销售指导价
  lastPurchasePrice?: number      // 最近采购价
  
  // 库存预警
  minStock: number                // 最低库存预警
  maxStock: number                // 最高库存
  
  // 关联信息
  defaultSupplierId?: number       // 默认供应商
  remarks: string
  
  createdAt: string
  updatedAt: string
}
```

### 3. 库位库存 (Stock) - 按仓库的实时库存

```typescript
interface Stock {
  id: number
  productId: number               // 商品ID
  warehouseId: number             // 仓库ID
  quantity: number                // 当前库存数量
  updatedAt: string
}
```

### 4. 出入库单据 (StockTransaction)

```typescript
type TransactionType = 
  | 'purchase_in'      // 采购入库
  | 'purchase_return'  // 采购退货
  | 'sale_out'         // 销售出库
  | 'sale_return'      // 销售退货
  | 'transfer_out'     // 调拨出库
  | 'transfer_in'      // 调拨入库
  | 'adjustment'       // 库存调整

interface StockTransaction {
  id: number
  transactionNo: string            // 单据编号
  type: TransactionType
  warehouseId: number             // 仓库ID
  
  // 商品信息
  productId: number
  quantity: number
  unitPrice: number
  totalAmount: number
  
  // 关联信息（可选）
  projectId?: number              // 关联项目（用于成本归集）
  contractId?: number             // 关联合同
  counterpartyId?: number          // 交易对方（供应商/客户）
  relatedTransactionId?: number    // 关联单据（如调拨出库的入库单）
  
  // 单据信息
  transactionDate: string
  operator: string                 // 经办人
  remarks: string
  
  createdAt: string
}
```

### 5. 库存汇总视图 (StockSummary) - 按商品汇总

```typescript
interface StockSummary {
  productId: number
  productCode: string
  productName: string
  category: string
  unit: string
  specifications: string
  
  // 各仓库库存
  warehouseStocks: {
    warehouseId: number
    warehouseName: string
    warehouseType: 'project' | 'company'
    quantity: number
  }[]
  
  // 汇总
  totalQuantity: number           // 总库存
  totalAmount: number             // 总金额（按最近采购价）
  minStock: number
  maxStock: number
  stockStatus: 'low' | 'normal' | 'over' | 'out'  // 库存状态
}
```

## 四、功能模块设计

### 模块一：仓库管理

```
仓库管理
├── 仓库列表（卡片视图）
│   ├── 项目仓库（按项目分组）
│   └── 公司仓库
├── 添加/编辑仓库
└── 仓库详情
    ├── 基本信息
    ├── 库存明细
    └── 出入库记录
```

**仓库类型区分**：
- **项目仓库**：自动创建/关联项目，数量变动影响项目成本
- **公司仓库**：商贸经营用，支持采购入库、销售出库

### 模块二：商品档案

```
商品档案
├── 商品列表（表格视图）
│   ├── 支持按分类、供应商筛选
│   ├── 显示各仓库库存汇总
│   └── 库存预警标识
├── 添加/编辑商品
├── 批量导入
└── 商品调价记录
```

**功能特点**：
- 唯一商品编码
- 多规格管理
- 历史价格追踪
- 供应商关联

### 模块三：出入库管理

```
出入库管理
├── 入库
│   ├── 采购入库（关联采购合同/供应商）
│   ├── 调拨入库（从其他仓库调入）
│   └── 退货入库（销售退货）
├── 出库
│   ├── 销售出库（关联销售合同/客户）
│   ├── 领料出库（关联项目，自动计入项目成本）
│   └── 调拨出库（调往其他仓库）
├── 库存调整（盘盈盘亏）
└── 单据列表
    ├── 支持按类型、日期、仓库筛选
    ├── 支持打印
    └── 支持导出Excel
```

### 模块四：库存查询

```
库存查询
├── 汇总视图
│   ├── 按商品汇总（各仓库库存一览）
│   ├── 按仓库汇总（各商品库存一览）
│   └── 按项目汇总（项目领料成本）
├── 明细视图
│   ├── 批次明细
│   └── 出入库流水
├── 预警中心
│   ├── 库存不足预警
│   ├── 积压预警
│   └── 效期预警（预留）
└── 统计分析
    ├── 出入库汇总报表
    ├── 商品出入库排行
    └── 库存周转分析
```

### 模块五：成本管理

```
成本管理
├── 项目材料成本
│   ├── 按项目统计领料金额
│   ├── 按时间趋势分析
│   └── 成本占比分析
├── 公司经营报表
│   ├── 采购统计
│   ├── 销售统计
│   └── 利润分析
└── 数据导出
```

## 五、页面布局设计

### 方案一：保持现状，入口整合

在侧边栏增加"仓库管理"入口，原有功能逐步迁移：

```
侧边栏菜单：
├── 项目管理
├── 任务管理
├── 员工管理
├── 成本管理        ← 原预算管理
├── 合同管理
├── 合作单位
├── 仓库管理        ← 新增（整合材料+进销存）
│   ├── 仓库列表
│   ├── 商品档案
│   ├── 出入库
│   ├── 库存查询
│   └── 成本报表
├── 发票管理
├── 结算办理
├── 数据看板
└── 系统设置
```

**迁移策略**：
1. 新增"仓库管理"模块，完全独立
2. 现有"材料管理"保留，逐步废弃
3. 现有"进销存"保留，逐步迁移
4. 提示用户使用新模块，旧模块数据可导入

### 方案二：彻底重构，合并入口

删除原有模块，全新设计仓库管理（改动较大，需迁移数据）

## 六、实施计划

### 第一阶段：基础搭建（1-2天）
1. 创建数据库表
   - `warehouses` - 仓库表
   - `products` - 商品档案表
   - `stocks` - 库位库存表
   - `stock_transactions` - 出入库单据表

2. 实现基础CRUD
   - 仓库管理（增删改查）
   - 商品档案（增删改查）
   - 基础库存查询

### 第二阶段：核心功能（2-3天）
1. 出入库流程
   - 入库单据（采购入库、调拨入库）
   - 出库单据（领料出库、销售出库、调拨出库）
   - 库存调整

2. 库存联动
   - 单据保存自动更新库存
   - 调拨生成一出一入两条记录

### 第三阶段：查询统计（1-2天）
1. 多维度库存查询
2. 出入库报表
3. 成本统计（按项目、按时间）

### 第四阶段：数据迁移（1天）
1. 现有材料管理数据导入
2. 现有进销存数据导入
3. 测试验证

### 第五阶段：界面优化（1天）
1. 响应式布局
2. 打印模板
3. 导出功能

## 七、关键设计决策

### 决策1：是否保留项目材料管理？

**建议**：废弃原有"材料管理"，统一使用"仓库管理"

**原因**：
- 现有材料管理功能过于简单（只有采购记录）
- 项目领料实际就是仓库出库
- 统一后数据结构更清晰

### 决策2：如何处理历史数据？

**迁移方案**：
1. `Material` 数据 → 转为 `Product` + 初始库存
2. `InventoryItem` 数据 → 转为 `Product`
3. `InventoryTransaction` 数据 → 转为 `StockTransaction`

### 决策3：是否支持多单位（采购单位 vs 使用单位）？

**暂不支持**，设计为单一单位，简化复杂度。如需支持，可在 `Product` 中增加 `purchaseUnit` 字段。

## 十、合同模板整合

### 设计决策
合同模板作为合同管理的子功能，不再单独作为菜单入口。

### 整合方案
在合同管理模块内增加"合同模板"Tab页：

```
合同管理
├── 合同看板
├── 收入合同
├── 支出合同
├── 其他协议     ← 框架/合作/赔偿等非收支合同
└── 合同模板    ← 移入合同管理
    ├── 模板列表
    ├── 上传模板
    ├── 模板变量设置
    └── 根据模板生成合同
```

### 实施改动
1. `Contracts.tsx` - 添加模板管理Tab
2. `App.tsx` - 移除"合同模板"菜单项
3. `ContractTemplates.tsx` - 保留作为独立组件，或合并到Contracts.tsx

| 风险 | 影响 | 对策 |
|------|------|------|
| 数据迁移丢失 | 高 | 迁移前完整备份，逐一验证 |
| 用户习惯改变 | 中 | 提供旧模块兼容，保持双入口一段时间 |
| 性能问题（大量库存查询） | 中 | 添加索引，按需查询而非全量加载 |
| 调拨流程不清晰 | 中 | 设计清晰的调拨审批流程 |

## 九、附录

### 字段说明

| 字段 | 说明 |
|------|------|
| transactionNo | 格式：RK-YYYYMMDD-序号（入库）、CK-YYYYMMDD-序号（出库） |
| stockStatus | out(缺货)、low(库存不足)、normal(正常)、over(积压) |
| projectId | 关联项目时，该笔出入库自动计入项目成本 |

### 报表设计

1. **库存汇总表**：商品编码、名称、分类、各仓库数量、库存状态
2. **入库明细表**：日期、单据号、商品、仓库、数量、单价、金额、供应商
3. **出库明细表**：日期、单据号、商品、仓库、数量、单价、金额、项目/客户
4. **项目成本表**：项目名称、商品分类、领料金额、占比
