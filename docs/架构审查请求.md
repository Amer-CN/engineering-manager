**当前状态**：主文件仍为 2338 行，与 features/members/ 下拆分组件并行存在

**问题**：
- 30+ 个 useState，6 个模态框
- OCR 逻辑、文件上传、表单处理全部混合
- features/members/ 已拆分组件，但未被主入口引用

**需要决策**：
- 是否让 Members.tsx 作为入口，逐步引用 features/ 下的组件？
- OCR 逻辑是否抽取为独立 hook（useIdCardOCR）？
- 文件上传逻辑如何统一？

#### 2. Invoices.tsx 拆分方案
**当前状态**：主文件为 1679 行

**问题**：
- 20+ 个 useState
- 发票 CRUD + 收款记录 CRUD + 两套文件上传 + 打印功能
- 拖拽/粘贴/预览逻辑重复

**需要决策**：
- 发票表单、收款表单是否拆分为独立组件？
- 打印功能（内联 HTML 模板）是否抽取为模板文件？
