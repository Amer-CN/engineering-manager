## 四、架构决策记录 (ADR)

### ADR-001: features/ 组件集成策略

**状态**: Accepted

**背景**:
开发团队在 features/ 目录下拆分了组件，但主入口仍在使用 monolithic 文件。

**决策**:
采用渐进式替换策略：
1. 主入口文件作为入口点
2. 逐步引用 features/ 下的组件
3. 清理重复代码后再删除 monolithic 组件

**后果**:
- ✅ 风险可控，每次变更影响范围小
- ✅ 可随时回滚
- ⚠️ 需要维护两套代码一段时间

---

### ADR-002: useFileUpload Hook 设计

**状态**: Proposed

**决策**:
创建统一的文件上传 Hook，支持：
- 拖拽上传
- 粘贴上传
- 点击上传
- 文件预览
- 统一的大小/类型限制

**接口设计**:

```typescript
interface UseFileUploadOptions {
  accept?: string[]           // ['image/jpeg', 'application/pdf']
  maxSize?: number            // 10 * 1024 * 1024
  onSuccess?: (file: UploadedFile) => void
  onError?: (error: string) => void
}

interface UseFileUploadReturn {
  files: UploadedFile[]
  isDragging: boolean
  preview: PreviewState | null
  addFile: (file: File) => void
  removeFile: (id: string) => void
  clearFiles: () => void
  openFileDialog: () => void
  getUploadProps: () => FileUploadProps  // 用于连接 UI 组件
}
```
